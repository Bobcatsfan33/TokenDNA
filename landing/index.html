Three things existing vendors don't catch

RSA 2026: every major identity vendor stopped at "did this agent authenticate?". TokenDNA picks up where they leave off.

Gap 1

Self-modification of policy

An agent silently expands its own permission scope mid-session. Your IAM policy looks unchanged; the agent now does things it never used to do. Caught the same week as the CrowdStrike F50 incident — by us.

Gap 2

Silent permission drift

Tools you allowed last quarter for one purpose are now being called for a different one. The auth event is identical; the behavioral profile has shifted 3×. Drift detector + trust graph anomaly fires before damage.

Gap 3

MCP chain pattern attacks

read_file("/etc/secrets") followed by send_email(to=external). Each call is allowed in isolation. The chain is exfiltration. Detected as read_then_exfil, blocked at the SDK boundary.

How it works

Every agent action flows through the same loop: detect → correlate → score → decide → enforce. p99 closes inside 100 ms.

1UIS eventAgent action emits a Universal Identity Schema event over HTTPS.
2Trust graphEdge added between agent and tool; cross-org edges checked against federation trusts.
3AnomalyDrift / scope-modification / chain-pattern detectors fire if the new edge violates baseline.
4Blast radiusIf anomalous, compute the set of downstream resources this agent can reach.
5Policy guardApply rules in order; ALLOW / STEP_UP / BLOCK / REVOKE verdict returned in <100 ms.
6Operatorpolicy_advisor synthesises a tightening suggestion the operator approves in one click.
